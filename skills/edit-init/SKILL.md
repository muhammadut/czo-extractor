Initialize CZO code editor for carrier: $ARGUMENTS

## What This Does

One-time setup per carrier. Creates a `.czo-workstreams/` workspace at the converter root with all paths, carrier metadata, and tool verification. Must be run before `/czo-extractor:edit-plan`.

## Step 1: Parse Arguments

- First word = carrier name (e.g., `Aviva`)
- Normalize capitalization to match folder names in `Companies/`

**If no carrier name is provided**, tell the user:
```
Usage: /czo-extractor:edit-init <CarrierName>

Example: /czo-extractor:edit-init Aviva

Available carriers:
```
Then list all carrier folders found under `*/Companies/` in the converter root and stop.

## Step 2: Auto-Detect Converter Root

Find the converter codebase by searching for the version folder structure:
1. Check current working directory for `v043/` folder
2. Search parent directories: `E:/cssi/Cssi.Net/Components/Cssi.Schemas/Cssi.Schemas.Csio.Converters`
3. The converter root is the directory containing `v043/`

If not found, tell the user to `cd` into the converter codebase first.

## Step 3: Auto-Detect VB Parser

Search these locations in order:
1. **Plugin bundled**: `find "$HOME/.claude/plugins/cache" -path "*/czo-extractor/*/tools/win-x64/vb-parser.exe" 2>/dev/null | head -1`
2. **Plugin dev directory**: Check `tools/win-x64/vb-parser.exe` relative to THIS plugin's root (the directory containing this SKILL.md's parent `skills/` directory)
3. **Converter root**: `<converter_root>/tools/win-x64/vb-parser.exe`
4. **System PATH**: `which vb-parser 2>/dev/null || where vb-parser 2>/dev/null`

If not found, error with install instructions.

## Step 4: Find Project File

Search for the .vbproj file:
```bash
ls <converter_root>/*.vbproj 2>/dev/null
```

There should be exactly one: `Cssi.Schemas.Csio.Converters.vbproj`. Store its absolute path.

If not found, error: "No .vbproj file found at converter root."

## Step 5: Find Carrier Version Folders

List all version folders that contain the carrier:
```bash
ls -d <converter_root>/*/Companies/<Carrier>/ 2>/dev/null
```

Sort by version number. Identify the **latest version** (highest number). Store the list.

Also check `ConverterFactoryLoader.vb` in the latest version to confirm the carrier is actively routed.

## Step 6: Check Extraction Status

Look for `.czo-extraction/carriers/<Carrier>/latest.json` at the converter root.

- If it exists: read `_metadata` to get extraction date, version info, code counts. Store in config.
- If it does NOT exist: **Print a warning** that extraction is needed. The `/czo-extractor:edit-plan` skill will handle running extraction automatically if needed.

## Step 7: Check ADO Credentials

Search for `.env` file with ADO credentials in this order:
1. `<converter_root>/.env`
2. `<converter_root>/../.env`
3. The known location: `E:/intelli-new/Cssi.Net/Portage Mutual/.env`

The `.env` must contain: `ADO_PAT`, `ADO_ORG`, `ADO_PROJECT`

If not found, tell the user to create `<converter_root>/.env`:
```bash
export ADO_PAT='your-personal-access-token'
export ADO_ORG='rivalitinc'
export ADO_PROJECT='Rival Insurance Technology'
export ADO_USE_VSCOM='1'
```

## Step 8: Create Workspace

Create `.czo-workstreams/` at the converter root if it doesn't exist.

### Write `paths.md`

```markdown
# CZO Editor Paths
<!-- Auto-generated by /czo-extractor:edit-init — do not edit manually -->

## Converter Codebase
- converter_root: <absolute path>
- vbproj: <absolute path to .vbproj>

## Carrier
- carrier_name: <Carrier>
- latest_version: <e.g., V148>
- version_folders: <comma-separated list>

## Tools
- vb_parser: <absolute path to vb-parser.exe>
- fetch_ticket: <absolute path to fetch-ticket.sh>
- plugin_root: <absolute path to this plugin>

## Credentials
- env_file: <absolute path to .env>

## Extraction
- extraction_root: <absolute path to .czo-extraction/>
- latest_json: <absolute path to latest.json, or "NOT_EXTRACTED">
- latest_rules: <absolute path to latest-rules.md, or "NOT_EXTRACTED">

## Workspace
- workstreams_root: <absolute path to .czo-workstreams/>
```

### Write `config.yaml`

```yaml
carrier:
  name: <Carrier>
  latest_version: <e.g., V148>
  versions:
    - folder: V132
      path: <absolute path>
    - folder: V134
      path: <absolute path>
    - folder: V148
      path: <absolute path>

key_files:
  company_constants: <path to latest CompanyConstants.vb>
  coverage_code_converter: <path to latest CoverageCodeConverter.vb>
  converter_factory_loader: <path to latest ConverterFactoryLoader.vb>
  enum_converter_factory: <path to latest EnumConverterFactory.vb>

extraction:
  exists: true/false
  date: <YYYY-MM-DD or null>
  total_codes: <count or null>
  z_codes: <count or null>

initialized: <ISO timestamp>
```

Print a summary:

```
Next: /czo-extractor:edit-plan <ticket-number>
```
